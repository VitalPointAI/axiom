Create `indexers/evm_fetcher.py` implementing `ChainFetcher` ABC:

1. **CHAIN_CONFIG** — Migrate from evm_indexer.py. Keep all 4 chains (ETH, Polygon, Cronos, Optimism). Keep `free_tier` and `custom_api` flags. Map chain names to lowercase for the `chain` column in transactions table: 'ethereum', 'polygon', 'cronos', 'optimism'.

2. **EVMFetcher class:**
   - `__init__(self, pool)` — call super().__init__(pool), set chain_name='evm', supported_job_types=['evm_full_sync', 'evm_incremental']
   - `sync_wallet(self, job)` — Main entry point from service.py:
     a. Extract wallet_address from job['account_id'], chain from job['chain']
     b. Look up CHAIN_CONFIG by chain name
     c. Determine start_block from job['cursor'] (or 0 for full_sync)
     d. Call `_fetch_all_tx_types(address, chain_config, start_block)` which calls normal, internal, ERC20, and NFT fetchers
     e. Transform each raw Etherscan result into a row matching the transactions table schema
     f. Bulk upsert using `psycopg2.extras.execute_values()` with `ON CONFLICT (chain, tx_hash, receipt_id, wallet_id) DO NOTHING`
     g. Update job cursor to `str(max_block_seen)`
     h. Update job progress_fetched

   - `_fetch_paginated(self, params, chain_config)` — NEW (not in evm_indexer.py):
     a. Set page=1, page_size=10000
     b. Loop: make request, check result count, increment page if count == page_size
     c. Use `offset` param for Etherscan V2 pagination: `offset=page_size, page=N`
     d. Rate limit: `time.sleep(0.25)` between requests (5 req/sec Etherscan limit)
     e. For Cronos, use chain_config['custom_api'] instead of ETHERSCAN_V2_URL
     f. Return all results concatenated

   - `_transform_tx(self, raw_tx, wallet_address, chain, tx_type)` — Transform raw Etherscan JSON to transactions table row:
     a. Direction: compare to/from with wallet_address (case-insensitive)
     b. For ERC20/NFT: tx_hash = f"{hash}-{logIndex}" to handle multiple transfers per parent tx
     c. For ERC20/NFT: receipt_id = original tx_hash (for cross-referencing)
     d. amount = int(value) stored as NUMERIC(40,0) — wei for native, token units for ERC20
     e. fee = int(gasUsed) * int(gasPrice) for normal tx only (internal/ERC20/NFT don't have separate gas)
     f. action_type = 'transfer' | 'internal' | 'erc20' | 'nft'
     g. token_id = contractAddress for ERC20/NFT, NULL for native
     h. raw_data = full Etherscan JSON as JSONB

   - `get_balance(self, address)` — Call Etherscan V2 `?module=account&action=balance` for native balance

3. **Tests** in `tests/test_evm_fetcher.py`:
   - Create `tests/fixtures/etherscan_responses.py` with sample Etherscan V2 JSON responses (normal tx, ERC20 transfer, internal tx)
   - Mock `requests.get` to return fixture data
   - Test transaction transformation, direction detection, fee calculation, pagination logic
   - Use `unittest.mock.patch` for API calls — do NOT require a database connection

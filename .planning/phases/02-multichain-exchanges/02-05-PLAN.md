class AIFileAgent:
    """AI-powered file ingestion using Claude API.

    Handles unknown/complex exchange export files that traditional
    parsers cannot detect. Extracts transaction data with confidence
    scores and flags uncertain records for human review.
    """

    def __init__(self, pool):
        self.pool = pool
        self._client = None  # Lazy init

    @property
    def client(self):
        if self._client is None:
            import anthropic
            self._client = anthropic.Anthropic()  # Uses ANTHROPIC_API_KEY env var
        return self._client

Key methods:

1. **`process_file(self, file_import_id: int, user_id: int) -> dict`** — Main entry point:
   - Load file from storage_path (from file_imports table)
   - Detect file type by extension: .csv, .xlsx, .pdf
   - Extract text content:
     - CSV: read directly as text
     - XLSX: use openpyxl to convert to CSV-like text (first sheet)
     - PDF: use pdfplumber to extract text
   - If file is too large (>100KB of text), chunk into sections
   - Call `_extract_transactions(text_content, filename)`
   - Insert extracted transactions into exchange_transactions
   - Return summary: {imported, flagged, errors, exchange_detected}

2. **`_extract_transactions(self, text: str, filename: str) -> List[dict]`** — Claude API call:
   - Build a system prompt that instructs Claude to:
     a. Identify the exchange/platform from the file content
     b. Extract every transaction as a JSON array
     c. For each transaction, output: tx_id, tx_date (ISO format), tx_type, asset, quantity, price_per_unit, total_value, fee, fee_asset, currency, notes, confidence (0.0-1.0)
     d. Set confidence < 0.8 for ambiguous rows (unclear type, missing amount, etc.)
   - Use `client.messages.create()` with model="claude-sonnet-4-20250514"
   - Parse the JSON response
   - Return list of transaction dicts

3. **`_insert_transactions(self, transactions: List[dict], user_id: int, exchange: str, file_import_id: int) -> dict`** — Database insertion:
   - For each transaction:
     - Generate tx_id if not present: `f"ai_{file_import_id}_{index}"`
     - Set source = 'ai_agent'
     - Set confidence_score from AI output
     - Set needs_review = True if confidence < CONFIDENCE_THRESHOLD
   - Use `ON CONFLICT (user_id, exchange, tx_id) DO NOTHING`
   - Use %s placeholders (psycopg2)
   - Return {imported, flagged, skipped}

4. **`_read_file_content(self, filepath: str) -> str`** — File reader:
   - .csv/.txt: open and read directly (with utf-8-sig encoding for BOM handling)
   - .xlsx: use openpyxl, read first sheet, convert rows to CSV-like text
   - .pdf: use pdfplumber, extract text from all pages
   - Truncate to 50,000 chars if needed (Claude context limit management)
   - Return text content

The system prompt for Claude should be specific:
```
You are a financial transaction extractor. Given the contents of an exchange or financial platform export file, extract all transactions into a structured JSON array.

For each transaction, provide:
- tx_id: The exchange's transaction/order ID (or generate one from date+type+amount if none visible)
- tx_date: ISO 8601 format (YYYY-MM-DDTHH:MM:SS)
- tx_type: One of: buy, sell, send, receive, deposit, withdrawal, staking_reward, interest, fee, transfer, trade, airdrop, mining, other
- asset: The crypto asset symbol (BTC, ETH, NEAR, etc.)
- quantity: The amount as a decimal string
- price_per_unit: Price per unit in fiat (null if not available)
- total_value: Total fiat value (null if not available)
- fee: Fee amount (null if none)
- fee_asset: Fee currency/asset (null if none)
- currency: Fiat currency (CAD, USD, EUR, etc.)
- notes: Any relevant context
- confidence: Your confidence in this extraction (0.0-1.0)

Set confidence below 0.8 for: ambiguous transaction types, missing critical fields, unclear amounts, possible duplicate rows, or unfamiliar formats.

Respond with ONLY a JSON object: {"exchange": "detected_name", "transactions": [...]}
```

Handle errors gracefully:
- If ANTHROPIC_API_KEY is not set, log error and return empty result
- If Claude returns invalid JSON, try to extract JSON from the response
- If file is unreadable, update file_imports with error message

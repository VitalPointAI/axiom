```python
class FileImportHandler:
    """Processes file_import jobs from the queue.

    Auto-detects exchange format, routes to appropriate parser,
    updates file_imports table with results.
    """

    def __init__(self, pool):
        self.pool = pool
        # Register all known parsers
        self.parsers = [
            CoinbaseParser(),
            CryptoComParser(),
            WealthsimpleParser(),
            GenericParser(),  # Handles Uphold, Coinsquare, Bitbuy
        ]

    def process_file(self, job: dict) -> None:
        """Process a file_import job.

        job['cursor'] contains the file_imports.id (set by upload API).

        Steps:
        1. Load file_imports record to get storage_path, user_id
        2. Read first 5 lines of file for header detection
        3. Try each parser's detect() method to find the right one
        4. If no parser matches, update file_imports.status = 'failed'
           with error 'Unknown exchange format — route to AI agent'
        5. If parser found, call parser.import_to_db(filepath, user_id, pool, batch_id)
        6. Update file_imports with results (rows_imported, rows_skipped, exchange_detected, status)
        """
```

Implementation details:
- Import all parsers: CoinbaseParser, CryptoComParser, WealthsimpleParser, GenericParser
- Get file_imports record: `SELECT * FROM file_imports WHERE id = %s`
- Update status to 'processing' at start
- Read first 5 lines using `open(path).readlines()[:5]`
- Loop through self.parsers, call `parser.detect(filepath, first_lines)`
- On match: call `parser.import_to_db(filepath, user_id, self.pool, batch_id=f"file_{file_import_id}")`
- On success: UPDATE file_imports SET status='completed', exchange_detected=parser.exchange_name, rows_imported=result['imported'], rows_skipped=result.get('skipped', 0)
- On error: UPDATE file_imports SET status='failed', error_message=str(error)
- If no parser matches: set status='needs_ai' (will be picked up by AI agent in plan 05)

Use `indexers.db` pool pattern — get conn from self.pool, execute queries, putconn in finally.

  <name>Task 3: Create file upload API endpoint</name>
  <files>web/app/api/upload-file/route.ts</files>
  <action>
Create `web/app/api/upload-file/route.ts` as a Next.js App Router POST handler:

1. Authenticate user via `getAuthenticatedUser()` from `@/lib/auth` (same pattern as other API routes)
2. Parse multipart form data using Next.js built-in `request.formData()`
3. Extract the file from form data
4. Generate SHA-256 hash of file contents for dedup
5. Check file_imports table: if `(user_id, file_hash)` already exists, return 409 Conflict with message "File already imported"
6. Save file to `uploads/{user_id}/{timestamp}_{filename}` directory
7. INSERT into `file_imports` table: user_id, filename, file_hash, file_size, mime_type, storage_path, status='queued'
8. Create a "virtual wallet" for exchange imports if not exists: INSERT INTO wallets (user_id, account_id, chain) VALUES (user_id, 'exchange_imports', 'exchange') ON CONFLICT DO NOTHING
9. Get the wallet_id for the exchange_imports wallet
10. INSERT into `indexing_jobs`: user_id, wallet_id (exchange_imports wallet), job_type='file_import', chain='exchange', status='queued', cursor=file_imports.id (so handler knows which file to process)
11. Update file_imports.job_id with the created job id
12. Return 201 with `{ file_import_id, job_id, filename, status: 'queued' }`

Error handling:
- 401 if not authenticated
- 400 if no file in request
- 409 if duplicate file hash
- 413 if file > 50MB
- 500 for database errors

Use parameterized queries ($1, $2) via the `db` helper from `@/lib/db` (TypeScript PostgreSQL client). Follow the same pattern as existing API routes (wallets, exchanges).

Create the `uploads/` directory path using `fs.mkdirSync` with `recursive: true`.

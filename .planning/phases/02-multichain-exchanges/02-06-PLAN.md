Create `indexers/dedup_handler.py` with `DedupHandler`:

The handler runs as a `dedup_scan` job type. It scans for potential duplicate transactions between on-chain data (transactions table) and exchange data (exchange_transactions table).

**Dedup algorithm:**
1. Query exchange_transactions that are NOT yet linked (no entry in tx_links for their id)
2. For each exchange transaction, search the transactions table for matches:
   - Same asset (exchange_transactions.asset maps to transactions.token_id or native chain token)
   - Amount within 1% tolerance (exchange quantities are in human-readable units; on-chain amounts are in wei/yocto — need conversion)
   - Timestamp within 10 minutes (exchange_transactions.tx_date vs transactions.block_timestamp converted to datetime)
   - Direction alignment: exchange 'send' matches on-chain 'out', exchange 'receive' matches on-chain 'in'
3. If match found, insert into a `tx_links` table (or add columns to exchange_transactions):
   - For simplicity, add `dedup_link_hash` VARCHAR(128) and `is_duplicate` BOOLEAN DEFAULT FALSE columns to exchange_transactions (via ALTER in the dedup handler, or handle gracefully if columns don't exist)
   - Actually, use a simpler approach: just set `needs_review = True` and `notes = 'Potential duplicate of on-chain tx {tx_hash}'` on the exchange_transactions row
   - This avoids schema changes and leverages existing columns

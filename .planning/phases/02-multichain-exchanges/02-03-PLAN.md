**Modify `indexers/exchange_parsers/base.py`:**

1. Remove `from db.init import get_connection` — replace with `from indexers.db import get_pool`
2. Make BaseExchangeParser inherit from ExchangeParser ABC (from indexers.exchange_plugin)
3. Update `import_to_db(self, filepath, user_id, pool, batch_id=None)`:
   - Accept `user_id` (int) and `pool` (SimpleConnectionPool) as required params
   - Generate batch_id if not provided: `datetime.now().strftime("%Y%m%d_%H%M%S")`
   - Call `self.parse_file(filepath)` to get transactions list
   - Get connection from pool: `conn = pool.getconn()`
   - For each transaction, execute:
     ```sql
     INSERT INTO exchange_transactions
         (user_id, exchange, tx_id, tx_date, tx_type, asset, quantity,
          price_per_unit, total_value, fee, fee_asset, currency,
          notes, raw_data, import_batch, source)
     VALUES (%s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s, 'csv')
     ON CONFLICT (user_id, exchange, tx_id) DO NOTHING
     ```
   - Use `%s` placeholders throughout (NEVER `?`)
   - Commit after all inserts
   - Return `{'imported': count, 'skipped': skipped, 'errors': len(self.errors), 'batch_id': batch_id}`
   - Put connection back: `pool.putconn(conn)` in finally block

4. Add `detect(self, filepath, first_lines)` method — default returns False, subclasses override

5. Ensure `parse_row()` returns `raw_data` as a dict (not string) so it serializes to JSONB correctly. Use `json.dumps()` only at the INSERT level if needed.

For ALL files: grep for `?` in SQL strings and replace with `%s`. Grep for `db.init` imports and replace with `indexers.db`. Add `import json` if needed for raw_data JSONB serialization.

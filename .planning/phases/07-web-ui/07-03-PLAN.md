From existing schema (indexing_jobs table - migration 001):
```sql
CREATE TABLE indexing_jobs (
    id SERIAL PRIMARY KEY,
    wallet_id INTEGER REFERENCES wallets(id),
    user_id INTEGER NOT NULL REFERENCES users(id),
    job_type VARCHAR(64) NOT NULL,
    status VARCHAR(32) NOT NULL DEFAULT 'queued',
    priority INTEGER DEFAULT 5,
    cursor TEXT,
    progress_fetched INTEGER,
    progress_total INTEGER,
    error_message TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    started_at TIMESTAMPTZ,
    completed_at TIMESTAMPTZ
);
```

1. **`api/schemas/wallets.py`:** Pydantic models:
   - WalletCreate: account_id (str), chain (str, default="NEAR")
   - WalletResponse: id, account_id, chain, sync_status, last_synced_at, created_at
   - SyncStatusResponse: wallet_id, stage (str), pct (int 0-100), detail (str), jobs (list of job summaries)

2. **`api/routers/wallets.py`:** Endpoints using `get_effective_user` dependency:
   - POST /api/wallets: INSERT wallet with ON CONFLICT(user_id, account_id, chain) DO NOTHING (return 409 if exists). Then queue pipeline jobs:
     - NEAR: full_sync (priority=10) + staking_sync (priority=8) + lockup_sync (priority=7)
     - EVM chains: evm_full_sync (priority=10)
     - classify_transactions auto-chains from handler (no explicit job needed)
   - GET /api/wallets: SELECT from wallets WHERE user_id=effective_user_id. For each wallet, derive sync_status using CASE subquery on indexing_jobs (same pattern as existing web/app/api/wallets/route.ts).
   - GET /api/wallets/{id}/status: Return pipeline stage progress using the stage mapping from RESEARCH.md. Query all indexing_jobs for wallet, determine current stage and percentage.
   - DELETE /api/wallets/{id}: Verify wallet.user_id matches effective user. DELETE wallet (CASCADE deletes related data).
   - POST /api/wallets/{id}/resync: Queue new pipeline jobs for existing wallet (re-index + reclassify + recalc).

   All DB calls wrapped with `await run_in_threadpool(...)`.

3. **Tests:** Mock DB pool. Test wallet creation job queuing, sync status calculation, user isolation, duplicate prevention.

  <action>
1. **`api/auth/session.py`:** Implement `create_session(user_id, response, conn)` — generates `secrets.token_hex(32)`, INSERT into sessions with 7-day expiry, `response.set_cookie("neartax_session", ...)` with httponly=True, secure=bool(os.environ.get("SESSION_SECURE","false")=="true"), samesite="lax". Implement `destroy_session(session_id, response, conn)` — DELETE from sessions, response.delete_cookie.

2. **`api/auth/passkey.py`:** Using py_webauthn library:
   - `start_registration(username, conn)`: call `webauthn.generate_registration_options()` with RP_ID and RP_NAME from env. Store challenge in challenges table with 60s expiry. Return options JSON + challenge_id.
   - `finish_registration(challenge_id, credential, conn)`: retrieve challenge from DB, call `webauthn.verify_registration_response()` with expected_challenge, expected_origin (ORIGIN env var), expected_rp_id. Create or find user (by username), INSERT passkey row. Return user_id.
   - `start_authentication(conn)`: call `webauthn.generate_authentication_options()` (discoverable credentials — no allowCredentials needed). Store challenge. Return options + challenge_id.
   - `finish_authentication(challenge_id, credential, conn)`: look up passkey by credential_id, verify with `webauthn.verify_authentication_response()`, update counter. Return user_id.

   Config: RP_ID from env (default "localhost"), RP_NAME from env (default "Axiom"), ORIGIN from env (default "http://localhost:3003").

3. **`api/auth/router.py`:** Mount all auth endpoints:
   - POST /auth/register/start — calls passkey.start_registration
   - POST /auth/register/finish — calls passkey.finish_registration + session.create_session
   - POST /auth/login/start — calls passkey.start_authentication
   - POST /auth/login/finish — calls passkey.finish_authentication + session.create_session
   - GET /auth/session — returns current user from get_current_user dependency
   - POST /auth/logout — calls session.destroy_session

   All DB calls wrapped with `run_in_threadpool()` to avoid blocking async event loop.

4. **Tests:** Use FastAPI TestClient. Mock py_webauthn functions to avoid real CBOR/COSE. Mock DB pool. Test session cookie attributes. Test challenge lifecycle (create, consume, expire).

1. **`api/auth/oauth.py`:**
   - `start_google_oauth(conn)`: generate state via `secrets.token_urlsafe(32)`, store in challenges table (type="oauth_state", 10min expiry). Return Google auth URL with PKCE params (client_id from GOOGLE_CLIENT_ID env, redirect_uri from OAUTH_REDIRECT_URI env, scope="openid email profile").
   - `finish_google_oauth(code, state, conn)`: verify state from challenges table, exchange code for tokens via httpx POST to `https://oauth2.googleapis.com/token`, fetch user info from `https://www.googleapis.com/oauth2/v3/userinfo`, upsert user by email (INSERT ON CONFLICT(email) DO UPDATE SET username=COALESCE(users.username, excluded.username)). Return user_id.

2. **`api/auth/magic_link.py`:**
   - `request_magic_link(email, conn)`: generate token via `itsdangerous.URLSafeTimedSerializer(SECRET_KEY).dumps(email)`, INSERT into magic_link_tokens with 15-min expiry. Send email via boto3 SES (mock in tests). Token contains email claim.
   - `verify_magic_link(token, conn)`: decode token with itsdangerous (max_age=900), check magic_link_tokens table for unused token, mark used_at=NOW(), upsert user by email, return user_id.

3. **Update `api/auth/router.py`:** Add endpoints:

   - POST /auth/oauth/callback — accepts {code, state}, returns session
   - POST /auth/magic-link/request — accepts {email}, returns {"sent": true}
   - GET /auth/magic-link/verify — accepts token query param, creates session, redirects to dashboard

4. **Tests:** Mock httpx for Google token exchange, mock boto3 SES for email sending. Test token expiry, reuse prevention, user upsert behavior.

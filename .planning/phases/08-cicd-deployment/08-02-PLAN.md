  <name>Task 1: Create GitHub Actions deploy workflow</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
Create `.github/workflows/deploy.yml` with these jobs:

**Trigger:**
- `on.push.branches: [main]` — auto-deploy on push to main
- `on.workflow_dispatch.inputs.rollback_sha` — manual trigger with optional rollback SHA

**Job 1: deploy**
- runs-on: ubuntu-latest
- Environment: production
- Steps:

1. **Checkout** — `actions/checkout@v4`

2. **Setup SSH key** — Write `${{ secrets.DEPLOY_SSH_KEY }}` to a temp file, chmod 600. Add server to known_hosts via `ssh-keyscan`.

3. **Create .env on server** — SSH to server and write `.env` file at DEPLOY_PATH with all secrets from GitHub Secrets:
   ```
   POSTGRES_PASSWORD=${{ secrets.POSTGRES_PASSWORD }}
   DATABASE_URL=postgres://neartax:${{ secrets.POSTGRES_PASSWORD }}@postgres:5432/neartax
   NEARBLOCKS_API_KEY=${{ secrets.NEARBLOCKS_API_KEY }}
   ALCHEMY_API_KEY=${{ secrets.ALCHEMY_API_KEY }}
   ETHERSCAN_API_KEY=${{ secrets.ETHERSCAN_API_KEY }}
   COINGECKO_API_KEY=${{ secrets.COINGECKO_API_KEY }}
   CRYPTOCOMPARE_API_KEY=${{ secrets.CRYPTOCOMPARE_API_KEY }}
   FASTNEAR_API_KEY=${{ secrets.FASTNEAR_API_KEY }}
   JOB_POLL_INTERVAL=5
   SYNC_INTERVAL_MINUTES=15
   NODE_ENV=production
   ```
   Use a heredoc in the SSH command. This ensures secrets are never committed to the repo.

4. **Deploy** — Run the deploy script:
   ```
   env DEPLOY_HOST=... DEPLOY_USER=... DEPLOY_PATH=... DEPLOY_SSH_KEY=... bash scripts/deploy.sh
   ```
   If `github.event.inputs.rollback_sha` is set, add `--rollback $SHA`.

5. **Health check** — SSH to server and run `bash scripts/healthcheck.sh`. If this fails, the workflow fails.

6. **Notify on failure** — Use `if: failure()` step to print deployment logs:
   ```
   SSH to server: docker compose -f docker-compose.prod.yml logs --tail=50
   ```

7. **Cleanup SSH key** — `if: always()` step to remove the temp SSH key file.

**Concurrency control:** Add `concurrency: group: deploy, cancel-in-progress: false` to prevent overlapping deployments.

**Important implementation details:**
- Use `${{ secrets.DEPLOY_SSH_KEY }}` as the private key CONTENT (not a file path) — write it to `~/.ssh/deploy_key` in the workflow
- SSH commands use `-o StrictHostKeyChecking=no` to avoid host key prompts
- The deploy script is run LOCALLY in the runner, and it SSHs to the server (script already handles this)
- Set `DEPLOY_SSH_KEY` env var to the temp key file path when calling deploy.sh

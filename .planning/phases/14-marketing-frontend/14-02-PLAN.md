         const apiUrl = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8000'
         
         const res = await fetch(`${apiUrl}/api/waitlist`, {
           method: 'POST',
           headers: { 'Content-Type': 'application/json' },

       }
     }

| Boundary | Description |

| Public internet -> POST /api/waitlist | Untrusted email input from anonymous users |
| Next.js proxy -> FastAPI backend | Internal network call, server-to-server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-04 | Spoofing | waitlist endpoint | mitigate | Rate limit 10/minute per IP via slowapi; prevents mass fake signups |
| T-14-05 | Tampering | email input | mitigate | Pydantic EmailStr validates email format on FastAPI; zod validates on client (Plan 03) |
| T-14-06 | Information Disclosure | waitlist response | accept | Response differentiates new vs duplicate email — acceptable for UX; email existence disclosure is low risk for a waitlist |
| T-14-07 | Denial of Service | waitlist endpoint | mitigate | slowapi rate limiting (10/min/IP) + PostgreSQL UNIQUE constraint prevents table bloat from repeated submissions |
| T-14-08 | Elevation of Privilege | waitlist endpoint | mitigate | Waitlist signup does NOT create a user account; separate table with no auth implications |
</threat_model>

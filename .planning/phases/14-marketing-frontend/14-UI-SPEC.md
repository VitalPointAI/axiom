## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding, badge padding |
| sm | 8px | Compact element spacing, button icon gap |
| md | 16px | Default element spacing, form field gap, card padding (mobile) |
| lg | 24px | Section padding (mobile), card padding (desktop), nav item gap |
| xl | 32px | Layout gaps, grid column gap |
| 2xl | 48px | Major section breaks between landing page sections |
| 3xl | 64px | Page-level vertical rhythm, hero top/bottom padding (mobile) |

Exceptions:
- Touch targets on mobile: minimum 44px height for all interactive elements (CTAs, nav items, theme toggle) per WCAG 2.5.5
- Hero headline leading on mobile: use tight line-height (1.1) to prevent overflow on small screens despite heading size
- Hero desktop padding: 96px — justified by viewport height proportion on 1280px+ screens. Applied as `py-24` (Tailwind `96px`) only on `xl:` breakpoint for the hero section wrapper.

## Typography

| Role | Size | Tailwind Class | Weight | Line Height | Usage |
|------|------|----------------|--------|-------------|-------|
| Body | 16px | `text-base` | 400 (regular) | 1.6 | Paragraph copy, feature descriptions, comparison table cells |
| Label / UI | 14px | `text-sm` | 400 (regular) | 1.4 | Nav links, badge text, form labels, caption text, footnotes |
| Heading | 24px → 32px | `text-2xl md:text-3xl` | 700 (bold) | 1.25 | Section headings (h2), feature card titles, pricing title |
| Display | 40px → 64px | `text-4xl md:text-6xl` | 700 (bold) | 1.1 | Hero headline, major section taglines |

Font family: `Inter, system-ui, sans-serif` — loaded via `next/font/google` (already in `web/app/layout.tsx`).

**Declared weights:** 400 (body copy, labels, UI text) and 700 (headings, display). Two weights total — sufficient contrast for the crypto-native aesthetic; Inter variable font makes this zero overhead.

**Declared sizes:** 4 canonical values — 14px, 16px, 24px, 40px. Responsive scaling (24→32 for Heading, 40→64 for Display) is an implementation detail expressed via Tailwind responsive classes, not additional declared sizes.

**Source:** Inter confirmed in layout.tsx. Sizes derived from mobile-first mandate (D-12, D-20) and crypto-native aesthetic (D-11) at discretion.

## Color

### Token Mapping (dark mode default per D-10)

Dark mode values are the primary rendering state. Light mode values are overrides when user selects light.

| Role | Dark Mode Value | Light Mode Value | Usage |
|------|----------------|------------------|-------|
| Dominant (60%) — background | `hsl(222.2 84% 4.9%)` — near-black navy | `hsl(0 0% 100%)` — white | Page background, section backgrounds |
| Secondary (30%) — surfaces | `hsl(217 33% 17%)` — deep slate | `hsl(0 0% 100%)` — white card | Cards, nav background, comparison table rows |
| Accent (10%) — brand | `#6366f1` (indigo-500) / gradient | `#4f46e5` (indigo-600) | Reserved exclusively for: primary CTA button, active nav indicator, gradient headline text, waitlist form submit, comparison table "Axiom" column header highlight |
| Accent gradient | `linear-gradient(135deg, #6366f1 0%, #8b5cf6 50%, #06b6d4 100%)` | same gradient | Hero headline text fill, hero background glow, section divider accents |
| Glow / ambient | `rgba(99, 102, 241, 0.15)` indigo glow | none | Hero background radial glow, feature card hover glow, geometric pattern overlays |
| Destructive | `hsl(0 62.8% 30.6%)` dark / `hsl(0 84.2% 60.2%)` light | — | Breach timeline incident markers (visual warning signal, not action destructive) |
| Muted text | `hsl(215 20.2% 65.1%)` dark / `hsl(215.4 16.3% 46.9%)` light | — | Subheadlines, secondary body copy, comparison table neutral values |
| Border | `hsl(217.2 32.6% 25%)` dark / `hsl(214.3 31.8% 91.4%)` light | — | Card borders, section dividers, table cell borders |

**Accent reserved for (explicit list):**
1. Primary CTA button ("Join the waitlist") — background fill
2. Hero headline gradient text fill
3. Active navigation item indicator
4. Comparison table "Axiom" column header background
5. Feature card icon backgrounds (small chip/badge, not full card)
6. Waitlist form submit button hover state

**Accent is NOT used for:** body text, secondary buttons (use ghost/outline variant), neutral badges, background fills, decorative borders.

**Source:** Existing CSS custom properties from `web/app/globals.css` for base tokens. Indigo-500/purple gradient is Claude's discretion per D-11 (crypto-native aesthetic, Uniswap/Phantom reference). Gradient chosen to differentiate from Phantom's solid purple while remaining in the crypto-native design space.

## Visual Hierarchy

**Hero focal point:** The hero headline is the primary visual anchor. On load, the eye path is: gradient headline text → subheadline → waitlist form → secondary CTA. All other elements (background glow, geometric patterns) are subordinate and must not compete with this path.

- Hero headline: Display role (text-4xl md:text-6xl, weight 700, gradient fill) — highest visual weight
- Hero subheadline: Body role (text-base, weight 400, muted text color) — deliberately lower weight than headline
- Waitlist form: Mid-weight — Input + CTA button grouped, accent-colored submit button draws the eye to the conversion action
- Background glow: `rgba(99, 102, 241, 0.15)` — ambient, never brighter than foreground content
- Geometric patterns: opacity 0.05–0.10 — texture only, not focal points

## Animation Contract

Source: Claude's discretion per CONTEXT.md. Framer Motion 12.38.0.

| Interaction | Animation | Duration | Easing |
|-------------|-----------|----------|--------|
| Section entrance (scroll into view) | `opacity: 0 → 1`, `y: 20 → 0` | 0.5s | `easeOut` |
| Feature card stagger | Each card delayed +0.1s from previous | 0.5s | `easeOut` |
| Hero entrance | `opacity: 0 → 1`, `y: 30 → 0` | 0.7s | `easeOut`, plays on mount |
| CTA button hover | Scale 1.02, glow intensity increase | 0.2s | `easeInOut` |
| Nav scroll state | Background opacity 0 → 0.9 (blur backdrop) | 0.3s | `easeOut` |
| Theme toggle | Icon swap | 0.2s | `easeInOut` |

**Reduced motion:** All Framer Motion animations must respect `prefers-reduced-motion`. Use `useReducedMotion()` hook from Framer Motion to disable transitions when user prefers reduced motion. This is non-negotiable for accessibility.

## Accessibility Contract

- Minimum contrast ratio: 4.5:1 for body text, 3:1 for large text (WCAG AA)
- Dark mode background (`hsl(222.2 84% 4.9%)`) + foreground (`hsl(210 40% 98%)`) passes AA at all body sizes
- Touch targets: minimum 44x44px on mobile for all interactive elements
- Framer Motion: `useReducedMotion()` gates all scroll animations — if true, render final state immediately with no transition
- Focus rings: use `ring-2 ring-indigo-500 ring-offset-2` on all focusable elements (override default shadcn ring to match accent color)
- Semantic HTML: `<nav>`, `<main>`, `<section aria-label="...">`, `<footer>` — not div soup
- Breach timeline links: `target="_blank" rel="noopener noreferrer"` on all external source links
- Images: `next/image` with `alt` text on all images; no decorative images without `alt=""`
- Theme Toggle: `aria-label="Toggle theme"` — required accessibility contract on the button element

  <action>
    Create `reports/export.py` with:

    1. `KOINLY_LABEL_MAP` dict mapping TransactionClassification.category values to Koinly labels. Use the mapping from RESEARCH.md.

    2. `KoinlyExport(ReportEngine)`:
       - `generate(self, user_id, tax_year, output_dir, year_end_month=12, full_history=False)`:
         - If not full_history: calls _check_gate, scopes to fiscal year
         - Queries transaction_classifications tc JOIN transactions t ON tc.transaction_id = t.id
           LEFT JOIN wallets w ON t.wallet_id = w.id
           WHERE tc.user_id = user_id AND tc.leg_type = 'parent'
         - ALSO queries transaction_classifications tc JOIN exchange_transactions et ON tc.exchange_transaction_id = et.id
           WHERE tc.user_id = user_id AND tc.leg_type = 'parent'
         - Maps each row to Koinly format:
           - Date: human-readable datetime from block_timestamp (NEAR: ns) or tx_date (exchange)
           - Sent/Received: based on direction + category
           - For swaps (category in swap/trade): look for child legs (sell_leg = sent, buy_leg = received)
           - Label: KOINLY_LABEL_MAP.get(category, '')
           - Description: f"{action_type or tx_type} - {account_id or exchange}"
           - TxHash: tx_hash or tx_id
         - NEAR amounts: stored as yoctoNEAR (Numeric(40,0)); convert to NEAR by dividing by Decimal('1000000000000000000000000')
         - EVM amounts: stored as wei; convert by dividing by Decimal('1000000000000000000')
         - Exchange amounts: already in human units
         - File: `koinly_export_{year}.csv` (year-specific) or `koinly_export_full.csv` (full history)
         - Returns dict: row_count, file_path

    Add TestKoinlyExport class to tests/test_reports.py. Mock cursor with sample classified transactions. Verify label mapping and amount conversion.
  </action>

         - Calls each export method

       - `_write_quickbooks_iif(self, output_dir, tax_year, gains_rows, income_rows)`:
         - File: `quickbooks_{year}.iif`
         - Header: `!TRNS\tDATE\tACCNT\tNAME\tAMOUNT\tDOCNUM\tMEMO`
         - For each disposal: TRNS line (Crypto Assets, proceeds), SPL line (Capital Gains, -gain_loss), ENDTRNS

       - `_write_sage50_csv(self, output_dir, tax_year, gains_rows, income_rows)`:
         - File: `sage50_{year}.csv`

         - Each event produces 2+ rows that sum to zero (debit = credit)
         - For disposal: Debit Cash/Proceeds, Credit Crypto Assets (ACB), Credit/Debit Capital Gains

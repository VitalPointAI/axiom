  <files>engine/fifo.py, tests/test_fifo.py</files>
  <behavior>
    - Test 1: Acquiring 10 units at $5 then 5 units at $10 creates two lots
    - Test 2: Disposing 8 units uses FIFO: first lot (10@$5, take 8) = cost $40
    - Test 3: Disposing more than oldest lot: exhausts first lot, takes from second
    - Test 4: Remaining lots after disposal reflect correct quantities
    - Test 5: get_holdings() returns list of remaining lots with quantity, cost_per_unit, total_cost
    - Test 6: get_cogs(year) returns total cost of lots disposed during the year
    - Test 7: FIFO produces different gain/loss than ACB average cost for same transactions
    - Test 8: Oversell (dispose more than held) handled gracefully with needs_review flag
  </behavior>
  <action>
    Create `engine/fifo.py` with `FIFOTracker`:

    1. `__init__(self)` — initializes per-token lot queues as dict[str, list[Lot]]. Each Lot = namedtuple('Lot', ['timestamp', 'units', 'cost_per_unit_cad', 'total_cost_cad']).

    2. `acquire(self, token_symbol, units, cost_per_unit_cad, timestamp)` — appends a new Lot to the token's queue. All values must be Decimal.

    3. `dispose(self, token_symbol, units, proceeds_per_unit_cad, timestamp)` — pops lots from front of queue (FIFO) until `units` are consumed. Returns list of LotDisposal = namedtuple('LotDisposal', ['lot', 'units_from_lot', 'cost_cad', 'proceeds_cad', 'gain_loss_cad', 'timestamp']). If queue exhausted before units consumed, returns partial with needs_review=True.

    4. `get_holdings(self, token_symbol=None)` — returns remaining lots for a token or all tokens. Each lot includes: token_symbol, units, cost_per_unit_cad, total_cost_cad, acquisition_timestamp.

    5. `get_total_cost(self, token_symbol=None)` — sum of total_cost_cad across all remaining lots.

    6. `replay_from_snapshots(self, rows)` — takes list of acb_snapshot-like rows (token_symbol, event_type, units_delta, cost_cad_delta/units_delta for cost_per_unit, block_timestamp) and replays them through acquire/dispose. This is how the report module feeds data to FIFO.

    IMPORTANT: All arithmetic uses Decimal. No float. Use `Decimal(str(value))` for any conversion.

    Create `tests/test_fifo.py` with TestFIFOTracker. Test with known numeric examples that demonstrate FIFO vs ACB differences.
  </action>

  </verify>
  <done>FIFOTracker maintains lot queues, disposes FIFO order, handles oversell gracefully. FIFO results differ from ACB average cost. All tests pass.</done>

         - Calls _check_gate
         - Queries latest acb_snapshot per token_symbol for the user (MAX(block_timestamp) per token):
           ```sql
           SELECT DISTINCT ON (token_symbol) token_symbol, units_after, acb_per_unit_cad, total_cost_cad
           FROM acb_snapshots WHERE user_id = %s
           ORDER BY token_symbol, block_timestamp DESC
           ```
         - For unrealized gain/loss: optionally accepts current_prices dict {token: price_cad} or queries price_cache for latest price. If no price available, leave unrealized columns blank.

    2. `COGSReport(ReportEngine)`:
       - `generate(self, user_id, tax_year, output_dir, year_end_month=12, tax_treatment='capital')`:
         - Calls _check_gate
         - Calculates:
           a. Opening inventory: sum of total_cost_cad from latest acb_snapshot per token BEFORE fiscal year start
           b. Acquisitions during year: sum of cost_cad_delta from acb_snapshots WHERE event_type='acquire' within fiscal year
           c. Closing inventory: sum of total_cost_cad from latest acb_snapshot per token at fiscal year end
           d. COGS = opening + acquisitions - closing
         - If tax_treatment='business_inventory': also compute FIFO-based COGS using FIFOTracker.replay_from_snapshots()
         - Writes `cogs_{year}.csv`:
           Headers: Token, Opening Inventory (CAD), Acquisitions (CAD), Closing Inventory (CAD), COGS (CAD), Method

           b. Capital gains (net): SUM(gain_loss_cad) from capital_gains_ledger for the tax year
           c. COGS: from COGSReport (if tax_treatment in ['business_inventory', 'hybrid'])

         - If tax_treatment='hybrid': generate two sections — capital property view (50% inclusion) and business inventory view (100% inclusion + COGS)

1. Create `indexers/price_service.py` with class `PriceService`:

   ```python
   class PriceService:
       def __init__(self, db_pool):
           self.db_pool = db_pool
           self.coingecko_api_key = COINGECKO_API_KEY
           self.cryptocompare_api_key = CRYPTOCOMPARE_API_KEY
   ```

   **Core method: `get_price(coin_id, date_str, currency="usd") -> Decimal`**
   - Check price_cache table first: `SELECT price FROM price_cache WHERE coin_id=%s AND date=%s AND currency=%s`
   - If cached, return immediately
   - If not cached:
     a. Fetch from CoinGecko: `GET /api/v3/coins/{coin_id}/history?date={dd-mm-yyyy}&localization=false`
        - Handle rate limits (30 calls/min free tier) with retry
        - Extract `market_data.current_price.{currency}`
     b. Fetch from CryptoCompare: `GET /data/pricehistorical?fsym={symbol}&tsyms={currency}&ts={timestamp}`
        - Map coin_id to symbol (e.g., "near" -> "NEAR", "ethereum" -> "ETH")
     c. If both prices available, check for outliers:
        - If |price_a - price_b| / min(price_a, price_b) > 0.5, use median (CoinGecko preferred as primary)
        - Otherwise average them
     d. If only one source available, use it
     e. Cache result in price_cache: `INSERT INTO price_cache ... ON CONFLICT ... DO UPDATE SET price = EXCLUDED.price, source = EXCLUDED.source`
   - Return the price as Decimal

   **get_price_batch(coin_id, start_date, end_date, currency="usd") -> dict[str, Decimal]:**
   - For date range, check cache first for all dates
   - Fetch missing dates from CoinGecko market_chart/range endpoint (more efficient than per-day)
   - Fill remaining gaps from CryptoCompare
   - Cache all results
   - Return dict of date_str -> price

   **get_cad_rate(date_str) -> Decimal:**
   - Stored as coin_id="usd", currency="cad" in price_cache
   - Fetch from exchangerate-api or CoinGecko if not cached

   **Module-level convenience function:**
   ```python
   _default_service = None
   def get_price(coin_id, date_str, currency="usd"):
       global _default_service
       if _default_service is None:
           _default_service = PriceService(get_pool())
       return _default_service.get_price(coin_id, date_str, currency)
   ```

  <files>indexers/staking_fetcher.py, indexers/lockup_fetcher.py</files>

1. Create `indexers/staking_fetcher.py` with class `StakingFetcher`:

   ```python
   class StakingFetcher:
       def __init__(self, db_pool, price_service):
           self.db_pool = db_pool
           self.price_service = price_service
           self.rpc_url = FASTNEAR_ARCHIVAL_RPC
   ```

   **Core method: `sync_staking(job_row)`**
   - Get wallet account_id and user_id from job
   - Find all validators this wallet has staked with:
     - Use NearBlocks staking deposits endpoint (via NearBlocksClient)
     - Also scan transactions for STAKE/UNSTAKE action types
   - For each validator:
     - Call `backfill_epoch_rewards(wallet_id, user_id, account_id, validator_id)`

   **backfill_epoch_rewards(wallet_id, user_id, account_id, validator_id):**
   - Find the earliest staking transaction for this wallet+validator
   - Get current epoch from RPC: `{"method":"status","params":[]}`
   - Find last snapshot epoch in DB: `SELECT MAX(epoch_id) FROM epoch_snapshots WHERE wallet_id=%s AND validator_id=%s`
   - For each epoch from (last_snapshot + 1) to current:
     - Query validator pool at that epoch via archival RPC:
       ```json
       {"method":"query","params":{"request_type":"call_function","account_id":"{validator_id}","method_name":"get_account","args_base64":"{base64(account_id)}","block_id":{epoch_block_height}}}
       ```
     - Parse response: extract staked_balance, unstaked_balance
     - Store epoch snapshot: INSERT INTO epoch_snapshots
     - Calculate reward: `reward = current_staked - previous_staked - deposits_in_epoch + withdrawals_in_epoch`
       - deposits_in_epoch: sum of STAKE transactions in this epoch window
       - withdrawals_in_epoch: sum of UNSTAKE transactions in this epoch window
     - If reward > 0:
       - Convert to NEAR (divide by 1e24)
       - Look up FMV: `self.price_service.get_price("near", epoch_date, "usd")` and "cad"
       - Insert staking_event with event_type='reward'
     - Update job progress
   - Handle RPC errors gracefully (some archival epochs may be unavailable — log and skip)

   **get_epoch_block_height(epoch_id) -> int:**
   - Use archival RPC to find the first block of an epoch
   - Cache locally (epochs don't change)

   **Notes on epoch timing:**
   - NEAR epochs are ~12 hours (43,200 blocks)
   - Epoch ID is available from validator info RPC call
   - Use archival RPC (`FASTNEAR_ARCHIVAL_RPC`) for historical data

2. Create `indexers/lockup_fetcher.py` with class `LockupFetcher`:

   ```python
   class LockupFetcher:
       def __init__(self, db_pool, price_service):
           self.db_pool = db_pool
           self.price_service = price_service
           self.rpc_url = FASTNEAR_RPC
   ```

   **Core method: `sync_lockup(job_row)`**
   - Get wallet details from job
   - Determine lockup account ID (typically `{hash}.lockup.near`)
   - If the wallet IS a lockup account, process directly
   - If the wallet has a linked lockup account, find it via:
     - Check transactions for interactions with *.lockup.near accounts
     - Or query known lockup factory contract

   **fetch_lockup_events(wallet_id, user_id, lockup_account_id):**
   - Fetch all transactions for the lockup account via NearBlocksClient
   - Parse each transaction for lockup-relevant methods:
     - `new` / creation → event_type='create'
     - `check_transfers_vote` → informational only (skip)
     - `transfer` → event_type='transfer' (tokens moved from lockup to owner)
     - `withdraw_from_staking_pool` → event_type='withdraw'
     - `deposit_to_staking_pool` → event_type='deposit' (to staking from lockup)
     - Vesting events: detect when vesting schedule completes
   - For each meaningful event:
     - Extract amount, timestamp, tx_hash
     - Look up FMV at time of event
     - Insert into lockup_events table: `INSERT ... ON CONFLICT DO NOTHING` (use tx_hash + event_type as uniqueness)

   **query_lockup_state(lockup_account_id) -> dict:**
   - RPC call to view lockup contract state:
     ```json
     {"method":"query","params":{"request_type":"call_function","account_id":"{lockup_account_id}","method_name":"get_balance","args_base64":"e30=","finality":"final"}}
     ```
   - Also call `get_vesting_information`, `get_locked_amount`, `get_liquid_owners_balance`
   - Return parsed state dict

   **Note:** Per CONTEXT.md, Aaron's lockup vesting is COMPLETE (~2021). The parser should handle both active and completed lockups, but focus on capturing historical events accurately.

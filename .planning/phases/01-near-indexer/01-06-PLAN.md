**Part A — Fix GET handler (NEAR wallets query, lines 26-42):**

Replace the current NEAR wallets query that references `indexing_progress` and `sync_status` with a query that derives sync status from `indexing_jobs`. Use a subquery or LEFT JOIN approach:

```typescript
const nearWallets = await db.all(`
  SELECT
    'near-' || w.id as id,
    w.id as raw_id,
    w.account_id as address,
    w.chain,
    w.label,
    COALESCE(
      CASE
        WHEN EXISTS (SELECT 1 FROM indexing_jobs ij WHERE ij.wallet_id = w.id AND ij.status = 'running') THEN 'syncing'
        WHEN EXISTS (SELECT 1 FROM indexing_jobs ij WHERE ij.wallet_id = w.id AND ij.status IN ('queued', 'retrying')) THEN 'pending'
        WHEN EXISTS (SELECT 1 FROM indexing_jobs ij WHERE ij.wallet_id = w.id AND ij.status = 'failed') THEN 'error'
        WHEN EXISTS (SELECT 1 FROM indexing_jobs ij WHERE ij.wallet_id = w.id AND ij.status = 'completed') THEN 'synced'
        ELSE 'pending'
      END
    ) as sync_status,
    (SELECT MAX(ij.completed_at) FROM indexing_jobs ij WHERE ij.wallet_id = w.id AND ij.status = 'completed') as last_synced_at,
    w.created_at,
    COALESCE(
      (SELECT SUM(ij.progress_fetched) FROM indexing_jobs ij WHERE ij.wallet_id = w.id AND ij.job_type = 'full_sync'),
      0
    ) as tx_count,
    'near' as wallet_type
  FROM wallets w
  WHERE w.user_id = $1
  ORDER BY w.created_at DESC
`, [auth.userId]);
```

Also remove `db.prepare(...).all()` syntax — the file uses two different db access patterns. Use the `db.all()` pattern (from the `db` import at line 3) which is the PostgreSQL-compatible pattern, matching `sync/status/route.ts`. Remove the `getDb()` call (line 23) and use the `db` import instead.

**Part B — Fix EVM and XRP wallet queries (lines 45-79):**

These queries reference `evm_wallets`, `evm_indexing_progress`, and `xrp_wallets` tables which are from the old schema and not in the new Alembic migration. Wrap each in a try/catch that returns an empty array on failure, so the GET handler doesn't crash if these old tables don't exist yet:

```typescript
let evmWallets: any[] = [];
try {
  evmWallets = await db.all(`
    SELECT
      'evm-' || w.id as id,
      w.id as raw_id,
      w.account_id as address,
      w.chain,
      COALESCE(w.label, w.account_id) as label,
      'pending' as sync_status,
      NULL as last_synced_at,
      w.created_at,
      0 as tx_count,
      'evm' as wallet_type
    FROM wallets w
    WHERE w.user_id = $1 AND w.chain != 'near'
    ORDER BY w.created_at DESC
  `, [auth.userId]);
} catch {
  // EVM wallets table may not exist yet (Phase 2)
}
```

Remove the separate `xrp_wallets` query entirely — XRP wallets will be stored in the unified `wallets` table with `chain = 'xrp'` in future phases. The query above already catches non-NEAR wallets.

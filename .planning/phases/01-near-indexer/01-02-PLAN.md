<task type="auto">
  <name>Task 1: Build Transaction Indexer</name>
  <files>indexers/near_indexer.py</files>
  <action>
Create the main NEAR transaction indexer:

1. Create `indexers/near_indexer.py` with:
   - `index_account(account_id, force_reindex=False)`:
     - Get wallet_id from wallets table (or create if missing)
     - Check last indexed timestamp to avoid re-indexing
     - Use NearBlocksClient.fetch_all_transactions() generator
     - For each transaction:
       - Determine direction (in if receiver=account_id, out if predecessor=account_id)
       - Extract counterparty (the other party)
       - Parse action type from actions array
       - Insert into transactions table (ON CONFLICT DO NOTHING by tx_hash)
     - Update wallet's last_indexed_at timestamp
   
   - `index_all_accounts()`:
     - Get all wallets from database
     - Index each sequentially with progress logging
     - Return summary (total txs, time taken)

2. Transaction parsing logic:
   - action_type: First action's "action" field (TRANSFER, FUNCTION_CALL, etc)
   - method_name: First action's "method" field (for FUNCTION_CALL)
   - amount: Sum of all actions' "deposit" field
   - fee: outcomes_agg.transaction_fee

3. Progress logging:
   - Log every 100 transactions
   - Log total count when done
   - Estimate remaining time based on rate
  </action>
  <verify>
```bash
cd projects/neartax
python3 -c "

print(f'Indexed {count} transactions')

```
  </verify>
  <done>Indexer successfully pulls all transactions for an account into database.</done>
</task>

<task type="auto">
  <name>Task 2: Build Balance Reconciliation</name>
  <files>verify/reconcile.py</files>
  <action>
Create balance reconciliation to verify indexer accuracy:

1. Create `verify/reconcile.py` with:
   - `get_onchain_balance(account_id)`:
     - Use FastNear RPC to get current NEAR balance
     - Return balance in NEAR (not yoctoNEAR)
   
   - `calculate_balance_from_txs(account_id)`:
     - Query all transactions for account from database
     - Sum: incoming transfers + received amounts
     - Subtract: outgoing transfers + fees
     - Handle staking separately (staked amounts don't reduce liquid balance directly)
     - Return calculated balance
   
   - `reconcile_account(account_id)`:
     - Get on-chain balance
     - Get calculated balance
     - Compare with tolerance (0.01 NEAR for rounding)
     - Return: {onchain, calculated, difference, match: bool}

2. Reconciliation report:
   - Print clear comparison table
   - Flag if difference > tolerance
   - Suggest investigation if mismatch

Note: Initial reconciliation may not match perfectly - staking complicates things.
We track this in Phase 3 (classification) and fix discrepancies in Phase 5.
  </action>
  <verify>
```bash
cd projects/neartax
python3 -c "
from verify.reconcile import reconcile_account
result = reconcile_account('vitalpointai.near')
print(f'On-chain: {result[\"onchain\"]:.4f} NEAR')
print(f'Calculated: {result[\"calculated\"]:.4f} NEAR')
print(f'Difference: {result[\"difference\"]:.4f} NEAR')
print(f'Match: {result[\"match\"]}')

```

<objective>
Build the standalone indexer service that polls a PostgreSQL job queue and a NEAR transaction fetcher that pulls complete transaction history via NearBlocks API with cursor-based resume.

Purpose: This is the core data ingestion pipeline. When a user adds a wallet, a job is queued. The indexer service picks it up, fetches all transactions via NearBlocks, stores them in PostgreSQL, and verifies completeness. The service is self-healing — failed jobs retry with exponential backoff until successful.

Output: A running indexer service (`indexers/service.py`) and NEAR chain handler (`indexers/near_fetcher.py`) that together fulfill DATA-01 (complete NEAR transaction history).
</objective>

       def __init__(self, db_pool):

   **Core method: `sync_wallet(job_row)`**
   - Takes a job row dict (from indexing_jobs table)
   - Gets wallet account_id from wallets table using job.wallet_id
   - If job.cursor is set, resume from there. Otherwise start fresh.
   - Calls `self.client.fetch_transactions(account_id, cursor, per_page=25)` in a loop
   - For each page:
     - Parse each transaction using `parse_transaction()`
     - Batch INSERT into transactions table using `INSERT ... ON CONFLICT (chain, tx_hash, receipt_id, wallet_id) DO NOTHING`
     - Update job: set cursor to next page cursor, increment progress_fetched
     - Commit after each page (crash-safe resume)
   - When no more pages (cursor is None), mark job as completed
   - Run verification checks

   **parse_transaction(raw_tx, wallet_id, user_id) -> dict:**

   - Determine direction: "in" if receiver matches account_id, "out" otherwise

   - For FUNCTION_CALL: extract method_name, args
   - For TRANSFER/STAKE: extract amount in yoctoNEAR
   - Extract fee (transaction_fee)
   - Extract block_height, block_timestamp
   - Store full raw transaction as JSONB in raw_data
   - Return dict matching Transaction model columns

  <name>Task 2: Build standalone indexer service with job queue polling</name>
  <files>indexers/service.py, indexers/Dockerfile, docker-compose.yml</files>
  <action>
1. Create `indexers/service.py` — the standalone indexer service:

   ```python
   class IndexerService:
       def __init__(self):
           self.pool = get_pool(min_conn=2, max_conn=5)
           self.handlers = {"near": NearFetcher(self.pool)}
           self.running = True
   ```

   **Main loop: `run()`**
   - `while self.running:` loop
   - Poll job queue: `SELECT * FROM indexing_jobs WHERE status IN ('queued', 'retrying') AND (next_retry_at IS NULL OR next_retry_at <= NOW()) ORDER BY priority DESC, created_at ASC LIMIT 1 FOR UPDATE SKIP LOCKED`
   - The `FOR UPDATE SKIP LOCKED` ensures only one worker grabs a job (future multi-worker ready)
   - If no job found, sleep for `JOB_POLL_INTERVAL` seconds, then check for incremental sync jobs to create (see below)
   - If job found:
     - Set status = 'running', started_at = now(), increment attempts
     - Dispatch to chain handler: `self.handlers[job.chain].sync_wallet(job)`
     - On success: set status = 'completed', completed_at = now()
     - On failure: catch exception, set status = 'retrying' if attempts < max_attempts else 'failed'
     - Calculate next_retry_at with exponential backoff: `now() + timedelta(seconds=min(300, 5 * (2 ** attempts)))` (caps at 5 min)
     - Store error in last_error
     - Self-healing: NEVER give up until max_attempts (set to 100 by default)

   **Incremental sync: `check_incremental_syncs()`**
   - Query wallets that have completed initial sync (have a 'completed' full_sync job)
   - Check if last sync was > SYNC_INTERVAL_MINUTES ago
   - If so, create a new 'incremental_sync' job for that wallet
   - Incremental sync uses the last known cursor from the most recent completed sync

   **Signal handling:**
   - Register SIGTERM and SIGINT handlers to set `self.running = False`
   - Graceful shutdown: finish current job before exiting

   **Entrypoint: `if __name__ == "__main__":`**
   - Parse optional CLI args: `--once` (process one job and exit, for testing)
   - Create IndexerService and call run()

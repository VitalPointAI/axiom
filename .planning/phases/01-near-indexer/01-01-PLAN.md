<task type="auto">
  <name>Task 1: Create PostgreSQL Schema</name>
  <files>db/schema.sql, db/init.py</files>
  <action>
Create PostgreSQL schema for storing NEAR transactions:

1. Create `db/schema.sql` with:
   - `wallets` table: id, account_id (unique), chain (default 'near'), label, is_owned, created_at
   - `transactions` table: id, tx_hash (unique), receipt_id, wallet_id (FK), direction (in/out), 
     counterparty_id, action_type (TRANSFER/STAKE/UNSTAKE/FUNCTION_CALL/etc), method_name,
     amount (NUMERIC 40,0 for yoctoNEAR), fee, block_height, block_timestamp, success, raw_data (JSONB)
   - `staking_events` table: id, wallet_id, validator_id, event_type (deposit/withdraw/reward),
     amount, block_timestamp, tx_hash (FK)
   - Indexes on wallet_id, block_timestamp, tx_hash

2. Create `db/init.py` with:
   - Function to create database if not exists
   - Function to run schema.sql
   - Connection pool setup using psycopg2

3. Create `config.py` with:
   - DATABASE_URL from env or default to localhost
   - NEARBLOCKS_BASE_URL = "https://api.nearblocks.io/v1"
   - RATE_LIMIT_DELAY = 0.1 (100ms between requests)

Use SQLite for local development, PostgreSQL for production (check DATABASE_URL prefix).
  </action>
  <verify>
```bash
cd projects/neartax
python3 -c "from db.init import init_db; init_db(); print('DB initialized')"
python3 -c "from config import DATABASE_URL; print(f'Using: {DATABASE_URL}')"
```
  </verify>
  <done>Database schema created, init script works, config loads correctly.</done>
</task>

<task type="auto">
  <name>Task 2: Build NearBlocks API Client</name>
  <files>indexers/nearblocks_client.py</files>
  <action>
Create NearBlocks API client with rate limiting and pagination:

1. Create `indexers/nearblocks_client.py` with:
   - `NearBlocksClient` class:
     - `__init__(self, base_url, rate_limit_delay=0.1)`
     - `_request(self, endpoint)` - handles rate limiting, retries on 429
     - `fetch_transactions(self, account_id, cursor=None, per_page=25)` - paginated tx fetch
     - `fetch_all_transactions(self, account_id)` - generator that yields all txs
     - `fetch_staking_deposits(self, account_id)` - get staking deposit summary
   
2. Response parsing:
   - Convert yoctoNEAR amounts to NEAR (divide by 1e24)
   - Parse block_timestamp to datetime
   - Extract action type, method name, deposit amount from actions array
   - Handle both incoming and outgoing transactions

3. Error handling:
   - Retry 3 times with exponential backoff on 429/5xx
   - Raise clear exceptions on 404 (account not found)
   - Log all requests for debugging

4. Test with vitalpointai.near:
   - Fetch first page of transactions
   - Print count and sample transaction
  </action>
  <verify>
```bash
cd projects/neartax
python3 -c "

print(f'Got {len(result[\"txns\"])} transactions')
print(f'First tx: {result[\"txns\"][0][\"transaction_hash\"]}')
"
```
  </verify>
  <done>NearBlocks client fetches transactions with rate limiting, pagination works.</done>

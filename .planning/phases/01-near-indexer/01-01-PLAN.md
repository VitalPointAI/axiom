2. Create `db/models.py` with SQLAlchemy 2.0 declarative models. Use `mapped_column()` style. Tables:

   **users** (preserve compatibility with existing web auth):
   - id: Integer, primary key, autoincrement (SERIAL)
   - near_account_id: String(128), unique, not null
   - created_at: DateTime(timezone=True), server_default=func.now()
   - last_login_at: DateTime(timezone=True), nullable

   **wallets**:
   - id: Integer, primary key (SERIAL)
   - user_id: Integer, ForeignKey("users.id"), not null, indexed
   - account_id: String(128), not null (unique per user via constraint)
   - chain: String(20), not null, default="near" (multi-chain ready)
   - label: String(256), nullable
   - is_owned: Boolean, default=True
   - created_at: DateTime(timezone=True), server_default=func.now()
   - UniqueConstraint("user_id", "account_id", "chain")

   **transactions**:
   - id: Integer, primary key (SERIAL)
   - user_id: Integer, ForeignKey("users.id"), not null, indexed
   - wallet_id: Integer, ForeignKey("wallets.id"), not null, indexed
   - tx_hash: String(128), not null
   - receipt_id: String(128), nullable
   - chain: String(20), not null, default="near"
   - direction: String(3), check in('in','out'), nullable
   - counterparty: String(128), nullable
   - action_type: String(64), nullable
   - method_name: String(128), nullable
   - amount: Numeric(40,0), nullable (yoctoNEAR or wei)
   - fee: Numeric(40,0), nullable
   - token_id: String(128), nullable (for FT transfers)
   - block_height: BigInteger, nullable
   - block_timestamp: BigInteger, nullable
   - success: Boolean, nullable
   - raw_data: JSONB, nullable (NOT TEXT — use JSONB per user decision)
   - created_at: DateTime(timezone=True), server_default=func.now()
   - UniqueConstraint("chain", "tx_hash", "receipt_id", "wallet_id")

   **indexing_jobs** (database-backed job queue per user decision):
   - id: Integer, primary key (SERIAL)
   - user_id: Integer, ForeignKey("users.id"), not null, indexed
   - wallet_id: Integer, ForeignKey("wallets.id"), not null
   - job_type: String(32), not null (e.g., "full_sync", "incremental_sync", "staking_sync", "lockup_sync")
   - chain: String(20), not null, default="near"
   - status: String(20), not null, default="queued", check in('queued','running','completed','failed','retrying')
   - priority: Integer, default=0 (higher = more urgent)
   - cursor: String(256), nullable (resume point for pagination)
   - progress_fetched: Integer, default=0
   - progress_total: Integer, nullable
   - attempts: Integer, default=0
   - max_attempts: Integer, default=100 (self-healing — many retries)
   - last_error: Text, nullable
   - started_at: DateTime(timezone=True), nullable
   - completed_at: DateTime(timezone=True), nullable
   - next_retry_at: DateTime(timezone=True), nullable
   - created_at: DateTime(timezone=True), server_default=func.now()
   - updated_at: DateTime(timezone=True), server_default=func.now(), onupdate=func.now()

   **staking_events**:
   - id: Integer, primary key (SERIAL)
   - user_id: Integer, ForeignKey("users.id"), not null, indexed
   - wallet_id: Integer, ForeignKey("wallets.id"), not null
   - validator_id: String(128), not null
   - event_type: String(20), check in('deposit','withdraw','reward')
   - amount: Numeric(40,0), nullable (yoctoNEAR)
   - amount_near: Numeric(24,8), nullable (human-readable NEAR)
   - fmv_usd: Numeric(18,8), nullable (fair market value at time)
   - fmv_cad: Numeric(18,8), nullable
   - epoch_id: BigInteger, nullable
   - block_timestamp: BigInteger, nullable
   - tx_hash: String(128), nullable
   - created_at: DateTime(timezone=True), server_default=func.now()

   **epoch_snapshots** (for epoch-level staking reward calculation):
   - id: Integer, primary key (SERIAL)
   - user_id: Integer, ForeignKey("users.id"), not null
   - wallet_id: Integer, ForeignKey("wallets.id"), not null
   - validator_id: String(128), not null
   - epoch_id: BigInteger, not null
   - staked_balance: Numeric(40,0), not null (yoctoNEAR)
   - unstaked_balance: Numeric(40,0), default=0
   - epoch_timestamp: BigInteger, nullable
   - created_at: DateTime(timezone=True), server_default=func.now()
   - UniqueConstraint("wallet_id", "validator_id", "epoch_id")

   **price_cache** (token-agnostic, supports all chains per user decision):
   - id: Integer, primary key (SERIAL)
   - coin_id: String(64), not null (e.g., "near", "ethereum", "bitcoin")
   - date: Date, not null
   - currency: String(10), not null (e.g., "usd", "cad")
   - price: Numeric(24,10), not null
   - source: String(32), nullable (e.g., "coingecko", "cryptocompare")
   - created_at: DateTime(timezone=True), server_default=func.now()
   - UniqueConstraint("coin_id", "date", "currency")

   **lockup_events**:
   - id: Integer, primary key (SERIAL)
   - user_id: Integer, ForeignKey("users.id"), not null
   - wallet_id: Integer, ForeignKey("wallets.id"), not null
   - lockup_account_id: String(128), not null
   - event_type: String(32), not null (e.g., "create", "vest", "unlock", "transfer", "withdraw")
   - amount: Numeric(40,0), nullable
   - amount_near: Numeric(24,8), nullable
   - fmv_usd: Numeric(18,8), nullable
   - fmv_cad: Numeric(18,8), nullable
   - block_timestamp: BigInteger, nullable
   - tx_hash: String(128), nullable
   - created_at: DateTime(timezone=True), server_default=func.now()

   - Create `db/migrations/alembic.ini` with sqlalchemy.url using env var pattern: `sqlalchemy.url = ${DATABASE_URL}`
   - Create `db/migrations/env.py` that:

   - Create `db/migrations/script.mako` (standard Alembic template)
   - Create `db/migrations/versions/001_initial_schema.py` as the initial migration that creates ALL tables above

4. The migration MUST use proper PostgreSQL types:
   - SERIAL (via autoincrement=True) NOT AUTOINCREMENT
   - NUMERIC NOT REAL/FLOAT
   - JSONB NOT TEXT for raw_data
   - TIMESTAMPTZ (DateTime with timezone=True) NOT TIMESTAMP
   - Use CHECK constraints for enum-like columns
   - Create indexes on all foreign keys and commonly queried columns (user_id, wallet_id, status, chain, block_timestamp)

5. Do NOT use `CREATE TABLE IF NOT EXISTS` — Alembic manages existence. Use `op.create_table()` in upgrade and `op.drop_table()` in downgrade.

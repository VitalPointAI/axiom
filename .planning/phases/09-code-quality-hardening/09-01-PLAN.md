  <files>.github/workflows/ci.yml, pyproject.toml, .github/workflows/deploy.yml</files>
  <action>
1. Create `.github/workflows/ci.yml` with a `test` job:
   - Trigger on `push: [main]` and `pull_request`
   - Uses `actions/checkout@v4` and `actions/setup-python@v5` with python 3.11
   - Install dependencies: `pip install -r requirements.txt ruff pytest-cov`
   - Lint step: `ruff check . --select E,W,F --ignore E501`
   - Test step: `pytest tests/ -q --tb=short -x` (no DATABASE_URL needed since all tests mock DB)
   - Coverage step: `pytest tests/ -q --cov=api --cov=engine --cov=indexers --cov=reports --cov-report=term-missing`

2. Create `pyproject.toml` with ruff config:
   ```toml
   [tool.ruff]
   line-length = 120
   target-version = "py311"

   [tool.ruff.lint]
   select = ["E", "W", "F"]
   ignore = ["E501", "E402", "W291", "W292", "W293", "E711", "E712"]
   ```
   Use generous ignores — this is an existing codebase, not greenfield. The goal is catching real bugs (F-rules: undefined names, unused imports) not style enforcement.

3. Modify `.github/workflows/deploy.yml` to depend on CI passing:
   - Change the deploy workflow trigger to `workflow_run` so it only deploys after CI passes:
     ```yaml
     on:
       workflow_run:
         workflows: ["CI Quality Gates"]
         branches: [main]
         types: [completed]
       workflow_dispatch:
         inputs:
           rollback_sha:
             description: "Commit SHA to rollback to (leave empty for normal deploy)"
             required: false
             default: ""
     ```
   - Add a condition to the deploy job: `if: github.event_name == 'workflow_dispatch' || github.event.workflow_run.conclusion == 'success'`
   - This ensures deploy only runs after CI succeeds, while preserving manual rollback capability.
  </action>

    - Test 1: decode_multi_hop_path with 2-token path returns [tokenA, tokenB]
    - Test 2: decode_multi_hop_path with 3-token path (A->B->C) returns [tokenA, tokenB, tokenC]
    - Test 3: decode_multi_hop_path with 4-token path returns all 4 addresses
    - Test 4: detect_swap returns hop_count=1 for standard 2-token swap
    - Test 5: detect_swap returns hop_count=2 for 3-token multi-hop, with token_path list
    - Test 6: detect_swap with non-exactInput selector returns hop_count=1 (backward compat)
    - Test 7: decode_multi_hop_path with malformed/short input returns empty list (no crash)
  </behavior>
  <action>
Add to EVMDecoder class in engine/evm_decoder.py:

1. `decode_multi_hop_path(self, path_bytes: bytes) -> list[str]`:
   - Uniswap V3 path encoding: [token_addr (20 bytes)][fee (3 bytes)][token_addr (20 bytes)]...
   - First token: path_bytes[0:20]
   - Each subsequent token: offset += 23 (20 addr + 3 fee), read next 20 bytes
   - Convert each 20-byte chunk to hex address with '0x' prefix and checksum
   - Return list of token addresses in hop order
   - Guard: if path_bytes length < 20, return empty list. If (len - 20) % 23 != 0, return partial (up to last valid token)

2. Update `detect_swap(self, tx: dict) -> dict`:
   - After detecting exactInput selector (0xc04b8d59), attempt to decode the path parameter from tx input data
   - ABI structure for exactInput: (bytes path, address recipient, uint256 deadline, uint256 amountIn, uint256 amountOutMinimum)
   - The path parameter is at offset 0 in the tuple, but ABI-encoded as dynamic bytes — read the offset pointer (first 32 bytes after selector), then length, then path bytes
   - Add `hop_count` to return dict: number of swaps = len(token_path) - 1
   - Add `token_path` to return dict: list of token addresses
   - For non-exactInput swaps, set hop_count=1 and token_path from existing tokens list
   - On decode failure (malformed data), fall back to existing behavior with hop_count=1

Write tests in tests/test_evm_decoder.py. Create synthetic input_hex values with known paths. For a 3-hop test: encode path as tokenA(20 bytes) + fee(3 bytes) + tokenB(20 bytes) + fee(3 bytes) + tokenC(20 bytes), then ABI-encode as bytes parameter of exactInput.

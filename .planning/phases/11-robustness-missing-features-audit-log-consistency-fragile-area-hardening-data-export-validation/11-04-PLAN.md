**ACB invariants in engine/acb/pool.py:**

1. Add a module-level function `check_acb_pool_invariants(pool, conn=None, user_id=None, context="")`:
   - Check: pool.total_cost_cad >= 0 (negative total cost = violation)
   - Check: pool.total_units >= 0 (negative units = violation)
   - Check: if pool.total_units > 0, pool.total_cost_cad should be >= 0 (cost without units = inconsistent)
   - On violations: log warning with pool.symbol and violation details
   - If conn is not None: call write_audit(conn, user_id=user_id, entity_type='acb_pool', action='invariant_violation', new_value={"symbol": pool.symbol, "violations": violations, "context": context}, actor_type='system')
   - Return True if no violations, False if violations found
   - Import write_audit from db.audit

2. In engine/acb/engine_acb.py, call check_acb_pool_invariants() after each acquire() and dispose() call during the replay loop. Pass the conn and user_id from the replay context. This is an in-memory check (no DB round-trip for the check itself — only the audit write if violation found).

**Classifier invariants in engine/classifier/writer.py:**

1. Add a function `check_classifier_invariants_batch(conn, user_id: int) -> list[dict]`:
   - Run a single GROUP BY query to find transactions with parent_count != 1:
     ```sql
     SELECT t.id, COUNT(tc.id) as parent_count
     FROM transactions t
     LEFT JOIN transaction_classifications tc
         ON tc.transaction_id = t.id AND tc.user_id = %s AND tc.leg_type = 'parent'
     WHERE t.user_id = %s
     GROUP BY t.id
     HAVING COUNT(tc.id) != 1
     LIMIT 100
     ```
   - For each violation: log warning, call write_audit with action='invariant_violation', entity_type='transaction_classification'
   - Return list of violation dicts [{transaction_id, parent_count}, ...]
   - This is called ONCE at the end of a classify_transactions job (in ClassifierHandler), NOT per-transaction

2. Also add a leg balance check: for swap classifications, verify sell + buy + fee legs exist and leg_index values are sequential. Run as a separate batch query on transactions WHERE category LIKE '%swap%'.

Add tests:
- test_acb_invariant_negative_balance: create pool, force negative via mock, verify check returns False and calls write_audit
- test_acb_invariant_clean_pool: normal pool, verify check returns True
- test_classifier_invariant_missing_parent: insert tx without classification, verify detected
- test_classifier_invariant_duplicate_parent: insert tx with 2 parent classifications, verify detected

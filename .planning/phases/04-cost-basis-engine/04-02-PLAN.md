**Rewrite `engine/acb.py`** completely. Remove legacy ACBTracker/PortfolioACB/check_superficial_loss. Replace with:

**ACBPool class** (pure in-memory, no DB):
- `__init__(self, symbol: str)` — symbol, total_units=Decimal("0"), total_cost_cad=Decimal("0")
- `acb_per_unit` property: total_cost_cad / total_units, quantized to Decimal("0.00000001") with ROUND_HALF_UP. Return Decimal("0") if total_units <= 0
- `acquire(self, units: Decimal, cost_cad: Decimal, fee_cad: Decimal = Decimal("0")) -> dict` — updates pool, returns snapshot dict with acb_per_unit, total_units, total_cost_cad
- `dispose(self, units: Decimal, proceeds_cad: Decimal, fee_cad: Decimal = Decimal("0")) -> dict` — calculates acb_used, net_proceeds, gain_loss_cad. If units > total_units: clamp to total_units and set needs_review=True. Returns snapshot dict with acb_used_cad, net_proceeds_cad, gain_loss_cad, acb_per_unit, total_units, needs_review

**TOKEN_SYMBOL_MAP** dict: mapping known contract addresses to canonical symbols:
- "wrap.near" -> "NEAR", "near" -> "NEAR"
- Common EVM contract addresses for USDC, USDT, WETH -> canonical symbols
- Exchange asset strings pass through unchanged (already "NEAR", "ETH", etc.)

**resolve_token_symbol(token_id, chain, asset=None) -> str** function:
- If asset is not None (exchange tx): return asset.upper()
- If token_id in TOKEN_SYMBOL_MAP: return mapped symbol
- If token_id is None and chain == "near": return "NEAR"
- If token_id is None and chain in ("ethereum","polygon","optimism","cronos"): return "ETH"
- Otherwise: return token_id or "UNKNOWN"

**NEAR_TIMESTAMP_DIVISOR = 10**9** — NEAR block_timestamp is nanoseconds; divide to get Unix seconds.

**normalize_timestamp(block_timestamp, chain) -> int** function:
- If chain == "near": return block_timestamp // NEAR_TIMESTAMP_DIVISOR
- Else: return block_timestamp (already seconds for EVM)

**ACBEngine class** (orchestrates full replay with DB persistence):
- `__init__(self, pool, price_service: PriceService)`
- `calculate_for_user(self, user_id: int) -> dict` — main entry point:
  1. Delete existing acb_snapshots, capital_gains_ledger, income_ledger for user_id (full replay from scratch per user decision)
  2. Query all classified transactions chronologically:
     ```sql
     SELECT tc.id, tc.category, tc.leg_type, tc.fmv_usd, tc.fmv_cad,
            tc.staking_event_id, tc.lockup_event_id, tc.parent_classification_id,
            tc.transaction_id, tc.exchange_transaction_id,
            t.block_timestamp AS t_block_timestamp, t.amount, t.fee, t.token_id, t.chain,
            et.asset, et.quantity, et.fee AS et_fee, et.timestamp AS et_timestamp,
            se.fmv_usd AS se_fmv_usd, se.fmv_cad AS se_fmv_cad, se.amount_near AS se_amount_near,
            le.fmv_usd AS le_fmv_usd, le.fmv_cad AS le_fmv_cad, le.amount_near AS le_amount_near
     FROM transaction_classifications tc
     LEFT JOIN transactions t ON tc.transaction_id = t.id
     LEFT JOIN exchange_transactions et ON tc.exchange_transaction_id = et.id
     LEFT JOIN staking_events se ON tc.staking_event_id = se.id
     LEFT JOIN lockup_events le ON tc.lockup_event_id = le.id
     WHERE tc.user_id = %s
       AND tc.category NOT IN ('spam', 'transfer', 'internal_transfer')
     ORDER BY COALESCE(t.block_timestamp, EXTRACT(EPOCH FROM et.timestamp)::BIGINT) ASC, tc.id ASC
     ```
  3. Group parent + child legs by parent_classification_id
  4. For each event group, determine operation type and call ACBPool.acquire or ACBPool.dispose
  5. After each operation, persist ACBSnapshot row via INSERT ON CONFLICT (user_id, token_symbol, classification_id) DO UPDATE
  6. For dispose events, call GainsCalculator.record_disposal()
  7. For income events (staking_reward, lockup_vest, airdrop), call GainsCalculator.record_income()
  8. Return stats dict: {snapshots_written, gains_recorded, income_recorded, tokens_processed}

**Event classification logic** within calculate_for_user:
- `category='income'` + staking_event_id: staking reward income. Use se_fmv_cad as cost, se_amount_near as units. Acquire into token pool.
- `category='income'` + lockup_event_id: lockup vesting income. Use le_fmv_cad as cost, le_amount_near as units. Acquire into token pool.
- `category='income'` without staking/lockup: general income (airdrop). Get price via price_service.get_price_cad_at_timestamp(). Acquire into pool.
- `category='capital_gain'` or `'capital_loss'` (parent with sell_leg + buy_leg): swap. sell_leg: dispose sold token. buy_leg: acquire received token at FMV. fee_leg: add fee to buy_leg ACB.
- `category='capital_gain'` or `'capital_loss'` (simple, no legs): simple sale. Dispose token.
- `category='fee'`: fee paid (gas). This is a disposal of the fee token at zero proceeds (cost only, no gain).
- Skip categories: 'spam', 'transfer', 'internal_transfer' (filtered in SQL)

**Token amount resolution:**
- On-chain tx: amount is in smallest unit (yoctoNEAR=1e24, wei=1e18). Convert to human units using chain-specific divisor.
- Exchange tx: quantity is already human-readable Decimal.
- NEAR divisor: Decimal("1e24"), EVM divisor: Decimal("1e18")

**FMV resolution priority:**
1. If tc.fmv_cad is not None: use directly (pre-captured by classifier)
2. If staking/lockup event: use se_fmv_cad/le_fmv_cad
3. Otherwise: call price_service.get_price_cad_at_timestamp(coin_id, unix_ts)

```python
"""Balance reconciliation engine for Axiom verification pipeline.

Reconciles calculated balances (ACBPool snapshots + raw tx replay) against
on-chain state for NEAR (decomposed: liquid + locked + staked) and EVM
(Etherscan V2 native balance). Exchange accounts support manual balance entry.

Auto-diagnoses discrepancies in 4 categories:
  1. missing_staking_rewards
  2. uncounted_fees
  3. unindexed_period
  4. classification_error
"""
```

**Class: BalanceReconciler(pool)**

Constructor takes psycopg2 pool. Import from config: FASTNEAR_RPC, RECONCILIATION_TOLERANCES. Import ETHERSCAN_V2_URL and CHAIN_CONFIG from indexers.evm_fetcher.

**Public method: reconcile_user(user_id: int) -> dict**
  - Query all wallets for user_id
  - Group wallets by chain
  - For each wallet: call _reconcile_wallet()
  - Return stats dict: {wallets_checked, within_tolerance, flagged, errors}

**Private: _reconcile_wallet(user_id, wallet_id, account_id, chain) -> None**
  - Get expected balance (dual cross-check):
    a. _get_acb_expected(user_id, token_symbol) -- latest ACBPool snapshot
    b. _get_replay_expected(user_id, wallet_id, chain) -- raw tx sum
  - Get actual balance (chain-specific):
    - chain == "near": _get_near_balance(account_id) returns (liquid, locked, staked, total)
    - chain in EVM chains: _get_evm_balance(account_id, chain) returns total
    - chain == "exchange": skip on-chain (actual_balance = manual_balance if set)
  - Compute difference = actual - expected_acb
  - Check tolerance: abs(difference) <= RECONCILIATION_TOLERANCES[chain]
  - If outside tolerance: run _auto_diagnose(user_id, wallet_id, chain, difference)
  - Upsert into verification_results (one row per wallet+token)
  - Use INSERT ... ON CONFLICT (wallet_id, token_symbol) DO UPDATE

**Private: _get_acb_expected(user_id, token_symbol) -> Decimal or None**
  ```sql
  SELECT units_after FROM acb_snapshots
  WHERE user_id = %s AND token_symbol = %s
  ORDER BY block_timestamp DESC, id DESC
  LIMIT 1
  ```
  Return Decimal units_after (the final pool balance for this token).
  NOTE: ACBPool is user-scoped (all wallets pooled), so for per-wallet reconciliation against on-chain, we need a different approach. For NEAR: the expected balance per wallet = sum of all in - sum of all out for that specific wallet from transactions table (the raw replay). The ACBPool expected is a user-level cross-check. Store both: expected_balance_acb (user-level per token) and expected_balance_replay (per-wallet from raw txs).

**Private: _get_replay_expected(user_id, wallet_id, chain) -> Decimal**
  For per-wallet expected balance via raw tx sum:
  ```sql
  SELECT
    COALESCE(SUM(CASE WHEN direction='in' THEN amount ELSE 0 END), 0) as total_in,
    COALESCE(SUM(CASE WHEN direction='out' THEN amount ELSE 0 END), 0) as total_out,
    COALESCE(SUM(CASE WHEN direction='out' THEN COALESCE(fee, 0) ELSE 0 END), 0) as total_fees
  FROM transactions
  WHERE wallet_id = %s AND chain = %s
  ```
  Convert from raw units (yoctoNEAR / wei) to human units using chain-appropriate divisor:
  - near: divide by 10^24
  - evm chains: divide by 10^18
  Return Decimal(total_in - total_out - total_fees) in human units.

**Private: _get_near_balance(account_id) -> tuple[Decimal, Decimal, Decimal, Decimal]**
  Three RPC calls:
  1. view_account for liquid (result["amount"]) and locked (result["locked"]) -- both yoctoNEAR
  2. Query staking_events for distinct validator_ids where this account has active stake:
     ```sql
     SELECT DISTINCT se.validator_id
     FROM staking_events se
     JOIN wallets w ON se.wallet_id = w.id
     WHERE w.account_id = %s AND se.event_type IN ('deposit', 'reward')
     ```
     Also use NearBlocks kitwallet staking endpoint as fallback for pre-indexing validators.
  3. For each validator: call get_account_staked_balance RPC (same pattern as staking_rewards.py get_pool_balance). Sum all staked amounts.
  Convert all from yoctoNEAR to NEAR (Decimal division by 10^24).
  Return (liquid, locked, staked, liquid+locked+staked).
  Wrap each RPC call in try/except; on error, store the error in rpc_error field and return what was obtainable.

**Private: _get_evm_balance(account_id, chain) -> Decimal**
  Call Etherscan V2: `module=account, action=balance, address=account_id, tag=latest`
  Use the chain's chainid from CHAIN_CONFIG. Need to map DB chain names back to CHAIN_CONFIG keys (reverse CHAIN_NAME_MAP).
  Use ETHERSCAN_API_KEY from os.environ.get("ETHERSCAN_API_KEY", "").
  Convert wei result to human units (Decimal / 10^18).
  On error: log and return Decimal(0) with rpc_error recorded.

**Private: _auto_diagnose(user_id, wallet_id, chain, difference) -> tuple[str, dict, Decimal]**
  Returns (diagnosis_category, diagnosis_detail, confidence).

  Run 4 heuristics in order, return first that matches with confidence > 0.5:

  1. **missing_staking_rewards** (NEAR only):
     - Count staking rewards in staking_events for this wallet
     - Count distinct epochs in epoch_snapshots where staked_balance > 0
     - If reward_count < epoch_count * 0.8: likely missing rewards
     - confidence: 0.70 if reward gap > 20%, 0.50 if > 10%
     - detail: {"reward_count": N, "expected_epoch_count": M, "gap_pct": X}

  2. **uncounted_fees**:
     - Sum fees from transactions table for this wallet
     - Compare to fee disposals tracked in ACB (join acb_snapshots where event relates to fee)
     - If difference ~ matches fee gap: likely uncounted fees
     - confidence: 0.75 if fee difference within 20% of balance difference
     - detail: {"total_fees_onchain": X, "total_fees_tracked": Y}

  3. **unindexed_period**:
     - Find max time gap in transactions.block_timestamp for this wallet
     - If any gap > 7 days (604800 seconds): potential unindexed period
     - confidence: 0.60
     - detail: {"gap_start": timestamp, "gap_end": timestamp, "gap_days": N}

  4. **classification_error**:
     - Count transfers to own wallets that are NOT classified as internal_transfer
     - If > 0: potential misclassification inflating outgoing balance
     - confidence: 0.65
     - detail: {"misclassified_transfer_count": N}

  If no heuristic matches: return ("unknown", {}, Decimal("0.0"))

**Private: _upsert_result(conn, result_dict) -> None**
  INSERT INTO verification_results (...) VALUES (...)
  ON CONFLICT (wallet_id, token_symbol) DO UPDATE SET
    expected_balance_acb = EXCLUDED.expected_balance_acb,
    expected_balance_replay = EXCLUDED.expected_balance_replay,
    actual_balance = EXCLUDED.actual_balance,
    difference = EXCLUDED.difference,
    ... (all updatable columns)
    verified_at = NOW(),
    updated_at = NOW()

**Important implementation notes:**
- Use Decimal throughout for all financial calculations (not float). Import from decimal.
- The legacy file uses `from db.init import get_connection` and `from indexers.near_indexer import get_wallet_id` -- these are DEAD imports from the SQLite era. Do NOT use them. Use the psycopg2 pool.
- For NEAR staked balance enumeration: query both staking_events table AND use the NearBlocks kitwallet/staking endpoint (via NearBlocksClient from indexers.nearblocks_client) to catch pre-indexing validators. But make the NearBlocks call optional (try/except) since it requires an API key.
- RPC timeout: 10 seconds per call (consistent with existing code).
- Log each wallet reconciliation result at INFO level.
- For exchange wallets (chain starts with 'exchange' or wallet has no on-chain presence): skip on-chain query, use manual_balance if it exists in verification_results already, otherwise mark as 'unverified'.

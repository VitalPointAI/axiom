```python
"""Missing transaction detector for Axiom verification pipeline.

Identifies time periods where indexed transactions don't account for on-chain
balance changes, using monthly checkpoint comparison against archival NEAR RPC.

When gaps are found:
  1. Records in verification_results with diagnosis='unindexed_period'
  2. Queues targeted re-index jobs for the gap period
  3. Re-index triggers the full pipeline cascade: fetch -> classify -> ACB -> verify
"""
```

**Class: GapDetector(pool)**

Constructor takes psycopg2 pool. Import FASTNEAR_ARCHIVAL_RPC, RECONCILIATION_TOLERANCES from config.

**Public method: detect_gaps(user_id: int) -> dict**
  - Get all NEAR wallets for user_id (gap detection is NEAR-only for now; EVM chains lack archival balance API in our stack)
  - For each NEAR wallet: call _detect_wallet_gaps()
  - Return stats: {wallets_scanned, gaps_found, reindex_jobs_queued}

**Private: _detect_wallet_gaps(user_id, wallet_id, account_id) -> list[dict]**
  1. Build balance series:
     ```sql
     SELECT block_timestamp, direction, amount, COALESCE(fee, 0) as fee
     FROM transactions
     WHERE wallet_id = %s AND chain = 'near'
     ORDER BY block_timestamp ASC
     ```
  2. Create monthly checkpoints. Group transactions by calendar month. For each month:
     - running_balance += sum(in amounts) - sum(out amounts) - sum(fees)
     - record checkpoint: {month, running_balance, last_block_height_in_month}
     - Use Decimal for running balance (convert from yoctoNEAR: Decimal(amount) / Decimal(10**24))
  3. For each checkpoint (limit to 24 per wallet to manage RPC calls -- last 2 years):
     - Query archival RPC for balance at that block height:
       ```python
       requests.post(FASTNEAR_ARCHIVAL_RPC, json={
           "jsonrpc": "2.0", "id": "1",
           "method": "query",
           "params": {
               "request_type": "view_account",
               "block_id": block_height,
               "account_id": account_id
           }
       }, timeout=15)
       ```
     - Extract amount (liquid balance only -- staked/locked changes are slower than monthly)
     - Convert from yoctoNEAR to NEAR (Decimal)
     - NOTE: archival balance is liquid only. The running balance from transactions includes everything. This is an approximation -- compare relative changes between checkpoints rather than absolute values:
       - delta_indexed = checkpoint[N].running_balance - checkpoint[N-1].running_balance
       - delta_onchain = archival[N] - archival[N-1]
       - gap = abs(delta_indexed - delta_onchain)
       - If gap > 2 * Decimal(RECONCILIATION_TOLERANCES["near"]): this month has missing txs

  4. For each identified gap:
     - Record gap period: start_block = checkpoint[N-1].last_block, end_block = checkpoint[N].last_block
     - Store in verification_results with:
       - diagnosis_category = 'unindexed_period'
       - diagnosis_detail = {"gap_start_block": X, "gap_end_block": Y, "gap_month": "YYYY-MM", "delta_indexed": str, "delta_onchain": str, "gap_amount": str}
       - diagnosis_confidence = 0.60 (archival balance is liquid only, not definitive)
       - status = 'open'
     - Queue targeted re-index job (see _queue_reindex)

  Return list of gap dicts.

**Private: _get_archival_balance(account_id, block_height) -> Decimal or None**
  Call FASTNEAR_ARCHIVAL_RPC with block_id = block_height. Return liquid balance in NEAR (Decimal). On error (timeout, rate limit, block not available): return None and log warning. Cache results in diagnosis_detail to avoid re-querying.
  Timeout: 15 seconds (archival is slower than regular RPC).

**Private: _queue_reindex(conn, user_id, wallet_id, start_block, end_block) -> None**
  Queue a targeted re-index job:
  ```sql
  INSERT INTO indexing_jobs (user_id, wallet_id, job_type, chain, status, priority, cursor)
  VALUES (%s, %s, 'full_sync', 'near', 'queued', 3, %s)
  ```
  cursor = str(start_block) -- NearFetcher uses cursor as the resume point.
  Priority 3 (moderate -- below regular sync at 5, but above background tasks).
  Check for existing pending job first (same dedup pattern as other handlers):
  ```sql
  SELECT id FROM indexing_jobs
  WHERE wallet_id = %s AND job_type IN ('full_sync','incremental_sync')
    AND status IN ('queued','running')
  ```
  Only queue if no pending job exists.
  Log: "Queued re-index for wallet_id={} blocks {}-{}"

**Important implementation notes:**
- Use Decimal throughout for balance calculations.
- Limit archival RPC calls to max 24 per wallet (monthly checkpoints for 2 years). This avoids rate limiting and keeps execution time reasonable.
- If a wallet has fewer than 2 months of transactions, skip gap detection (insufficient data).
- Handle archival RPC errors gracefully: if a specific block height query fails, skip that checkpoint and continue. Don't fail the entire scan.
- EVM gap detection is out of scope for Phase 5 (no archival balance API in our Etherscan integration). Log a note for EVM wallets: "Gap detection not available for EVM chains in Phase 5."
- The re-index cascade (fetch -> classify -> ACB -> verify) happens automatically because NearFetcher completion triggers DedupHandler -> ClassifierHandler -> ACBHandler -> VerifyHandler chain.

```python
def run_verify(self, job: dict) -> None:
    """Run full verification suite for a user."""
    from verify.reconcile import BalanceReconciler
    from verify.duplicates import DuplicateDetector
    from verify.gaps import GapDetector
    from verify.report import DiscrepancyReporter

    user_id = job["user_id"]
    logger.info("Starting verification for user_id=%s", user_id)

    # Step 1: Balance reconciliation
    reconciler = BalanceReconciler(self.pool)
    reconcile_stats = reconciler.reconcile_user(user_id)
    logger.info(
        "Reconciliation complete for user_id=%s: %d wallets checked, "
        "%d within tolerance, %d flagged, %d errors",
        user_id,
        reconcile_stats.get("wallets_checked", 0),
        reconcile_stats.get("within_tolerance", 0),
        reconcile_stats.get("flagged", 0),
        reconcile_stats.get("errors", 0),
    )

    # Step 2: Duplicate detection
    detector = DuplicateDetector(self.pool)
    dedup_stats = detector.scan_user(user_id)
    logger.info(
        "Duplicate scan complete for user_id=%s: %d hash dupes merged, "
        "%d bridge flagged, %d exchange flagged/merged",
        user_id,
        dedup_stats.get("hash_dupes_merged", 0),
        dedup_stats.get("bridge_flagged", 0),
        dedup_stats.get("exchange_flagged", 0) + dedup_stats.get("exchange_merged", 0),
    )

    # Step 3: Gap detection (NEAR only)
    gap_detector = GapDetector(self.pool)
    gap_stats = gap_detector.detect_gaps(user_id)
    logger.info(
        "Gap detection complete for user_id=%s: %d wallets scanned, "
        "%d gaps found, %d re-index jobs queued",
        user_id,
        gap_stats.get("wallets_scanned", 0),
        gap_stats.get("gaps_found", 0),
        gap_stats.get("reindex_jobs_queued", 0),
    )

    # Step 4: Update account verification status rollup
    self._update_account_status(user_id)

    # Step 5: Generate discrepancy report
    reporter = DiscrepancyReporter(self.pool)
    report_path = reporter.generate_report(user_id)
    logger.info("Discrepancy report written to %s", report_path)

    logger.info(
        "Verification complete for user_id=%s: "
        "reconciled=%d, dupes=%d, gaps=%d",
        user_id,
        reconcile_stats.get("wallets_checked", 0),
        dedup_stats.get("hash_dupes_merged", 0) + dedup_stats.get("exchange_merged", 0),
        gap_stats.get("gaps_found", 0),
    )
```

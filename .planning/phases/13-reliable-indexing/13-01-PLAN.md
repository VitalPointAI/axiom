  <action>
Create Alembic migration 011 that creates:

1. `api_cost_log` table:
   - id BIGSERIAL PRIMARY KEY
   - logged_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - chain TEXT NOT NULL (e.g., 'near', 'ethereum', 'polygon')
   - provider TEXT NOT NULL (e.g., 'neardata_xyz', 'nearblocks', 'etherscan', 'alchemy')
   - call_type TEXT NOT NULL (e.g., 'block_fetch', 'wallet_txns', 'balance_check')
   - response_ms INT
   - estimated_cost_usd NUMERIC(12, 8) DEFAULT 0
   - Index: ix_api_cost_log_chain_date ON (chain, logged_at DESC)

2. `chain_sync_config` table:
   - chain TEXT PRIMARY KEY
   - enabled BOOLEAN NOT NULL DEFAULT true
   - fetcher_class TEXT NOT NULL (e.g., 'NearStreamFetcher', 'EVMStreamFetcher')
   - job_types TEXT[] NOT NULL (PostgreSQL array of job type strings)
   - config_json JSONB NOT NULL DEFAULT '{}'
   - monthly_budget_usd NUMERIC(8,2)
   - created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

3. `api_cost_monthly` view:
   - SELECT chain, provider, call_type, date_trunc('month', logged_at) AS month, COUNT(*) AS call_count, SUM(estimated_cost_usd) AS total_cost_usd FROM api_cost_log GROUP BY 1, 2, 3, 4

4. Seed chain_sync_config with initial rows for existing chains:
   - near: fetcher_class='NearStreamFetcher', job_types=['near_stream_sync','full_sync','incremental_sync','staking_sync','lockup_sync'], config_json={'poll_interval': 0.6, 'provider': 'neardata_xyz', 'historical_provider': 'nearblocks'}

   - optimism: chain_id=10
   - cronos: chain_id=25
   - xrp: fetcher_class='XRPFetcher', job_types=['xrp_full_sync','xrp_incremental']
   - akash: fetcher_class='AkashFetcher', job_types=['akash_full_sync','akash_incremental']

Follow the existing Alembic migration pattern in 010_onboarding_columns.py. Use op.execute() for raw DDL since this is a new table (not altering existing models).

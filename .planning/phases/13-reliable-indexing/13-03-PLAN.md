  <action>
Create `indexers/evm_stream_fetcher.py`:

1. Class `EVMStreamFetcher(ChainFetcher)`:
   - __init__(self, pool, cost_tracker=None) — pool for DB, optional CostTracker
   - Holds reference to EVMFetcher instance for historical sync delegation
   - PING_INTERVAL = 20 seconds
   - WATCHDOG_TIMEOUT = 60 seconds
   - MAX_BACKOFF = 60 seconds

2. WebSocket streaming:
   - `async watch_blocks(ws_url, on_new_head)` — connect via `websockets` library, send eth_subscribe("newHeads"), process incoming messages. On connection drop, reconnect with exponential backoff. Reset backoff on successful connection.
   - `get_ws_url(chain, config_json)` — construct WebSocket URL from ALCHEMY_API_KEY env var. Support chains: ethereum (wss://eth-mainnet.g.alchemy.com/v2/{key}), polygon (wss://polygon-mainnet.g.alchemy.com/v2/{key}), optimism (wss://opt-mainnet.g.alchemy.com/v2/{key}). For Cronos: use Infura or skip WebSocket (Cronos not on Alchemy). Return None if no API key configured.
   - Watchdog: track `last_message_time`. If current_time - last_message_time > WATCHDOG_TIMEOUT, raise ConnectionError to trigger reconnect.

3. ChainFetcher ABC implementation:
   - `sync_wallet(self, job)` — for historical sync (job_type evm_full_sync), delegate to existing EVMFetcher.sync_wallet(). Record last_block_number in json_cursor for streaming pickup.
   - `get_balance(self, address)` — delegate to existing EVMFetcher.get_balance()

4. On new block callback:
   - `async on_new_block(block_header, chain, tracked_wallets)` — when new block arrives, queue an incremental sync job for wallets on this chain. The actual transaction fetch is handled by EVMFetcher's Etherscan V2 pagination with startblock parameter.

5. Error handling:
   - Log all WebSocket disconnections with chain + error
   - Exponential backoff: 1, 2, 4, 8, 16, 32, 60, 60, ... seconds
   - If ALCHEMY_API_KEY not set, log warning and skip WebSocket (fallback to polling)
   - If cost_tracker provided, track WebSocket connection events

Note: Do NOT import web3.py. Use raw `websockets` library for eth_subscribe — lighter weight and avoids web3.py's heavy dependency tree. The research suggested web3.py but raw websockets is simpler for just newHeads subscription.

Write tests in `tests/test_evm_stream_fetcher.py`:

- Test get_ws_url returns correct URLs per chain
- Test that missing ALCHEMY_API_KEY returns None from get_ws_url
- Use unittest.mock.AsyncMock and asyncio test patterns
